# RaccoonClaw-OSS v0.1.0

## Summary

This is the first community-edition baseline release of the OpenClaw workbench. The focus is not feature breadth; it is getting a local-first, repo-local, observable task workbench into a state that a new contributor can boot from a clean checkout and reason about safely.

## Included In Community Edition

- chat with the chief of staff
- direct / light / full routing
- one-shot task creation
- task status board
- delivery archive
- model and skill configuration
- template-driven task kickoff

## Changed

- the primary startup path is now `local install + repo-local .openclaw`
- `clean` and `demo` seed profiles are part of the documented baseline
- optional modules are feature-flagged instead of assumed on by default

## Fixed

- runtime data paths no longer silently fall back to personal `~/.openclaw` state in the main community flow
- scheduled task cancellation now disables the underlying cron job instead of allowing silent re-registration
- delivery archive search and scheduled-task UI paths are covered by browser smoke and migration regressions

## Optional Features

- IM channels: off by default
- Toolbox / gateway controls: off by default
- Scheduled tasks: opt-in
- Automation mirrors: off by default
- Docker / Compose: optional enhancement, not the main support contract

## Upgrade Notes

- existing `JJC-*` data is still supported through migration and compatibility tests
- if you are moving from a personal runtime, prefer a fresh repo-local `.openclaw` seed before re-importing old data
- optional features should be enabled explicitly through `.env`

## Verification

- backend regressions: passed
- frontend build: passed
- browser smoke: passed
- migration regressions: passed
- Docker smoke: optional

## Known Limits

- community edition is still single-machine first
- advanced integrations remain more environment-sensitive than the core workflow
- Docker is maintained as a convenience path, not the primary release gate
